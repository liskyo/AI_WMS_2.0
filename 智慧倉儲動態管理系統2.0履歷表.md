## 🌟 亮點精華版 (適合 104 專案介紹 / 履歷表限制字數區塊)

**【專案名稱】** 智慧倉儲動態管理系統 2.0 (獨立全端開發)
**【技術棧】** React, Vite, TailwindCSS, Node.js, Express, SQLite, Axios, framer-motion, html5-qrcode
**【專案簡介】**
為解決傳統工廠倉儲「找料大海撈針」、「帳物不符」與「異質系統(ERP)斷層」三大痛點，從零到一獨立架構成的現代化 Web WMS 系統。採用前後端分離架構，實現跨裝置相機掃描、動態 2D 疊貨地圖、自動化低庫存採購報表，並打通與舊型 ERP 的檔案交換非同步機制，徹底根除紙本作業，提升整體倉儲吞吐效率達 50% 以上。

**【核心貢獻與亮點】**
1. **獨創 2D 動態地圖與複合疊貨引擎 (Visual WMS)**：突破傳統文字條列介面，利用 React `useMemo` 與動態 CSS Grid，將複雜座標（含不可通行障礙物、大型通道）轉化為具備「閃爍導航 (Ping)」的互動地圖。並實作高階「同點位多物件疊貨聚合標籤」，徹底消滅儲位顯示死角。
2. **異質系統高容錯非同步整合 (ERP Backend Integration)**：為對接公司內部無 API 介面的傳統 ERP，實作 Node.js `chokidar` 檔案監聽引擎 (Hot Folder Pattern)。將 ERP 匯出的 Excel 自動轉化為 WMS 待辦單據，並將派發結果精準轉為 ERP 定規匯入檔，雙向系統完全解耦、死機互不干涉。
3. **行動端 PDA 跨平台解耦化 (Mobile Scanner)**：整合 WebRTC 相機 API (`html5-qrcode`) 與 Local HTTPS 憑證，破解手機鏡頭權限封鎖。深度重構 React 生命週期，徹底消滅第三方相機資源與組件掛載產生的競爭崩潰 (Race Condition)，將員工自有手機穩定轉化為零成本的高效掃描槍。
4. **巨量數據覆蓋與 ACID 交易安全**：針對工廠動輒數萬筆的初期盤點與地形格網建立，棄用沉重的關聯式資料庫，改採 Node.js 非同步串接 `better-sqlite3`。結合 `Transaction` 交易保護與「完全覆蓋軟刪除演算法」，達成破萬筆多表資料的秒級無損同步與防呆回滾。
5. **智能安全庫存與主動式採購決策**：首創全系統聯動的低庫存預警聯動機制。實際庫存一低於安全閥值，首頁即刻觸發動態紅燈警告，點擊後系統自動跨頁生成「建議採購清單 Excel 報表」並一鍵下載，顛覆以往被動查詢的生管模式。

## 🚀 深度技術剖析版 (適合面試官深挖 / 面試簡報 / GitHub Readme)

若進入技術面試，你可以針對不同環節展示工程師底蘊：

### 1. 異質系統的高容錯非同步對接與職責分離 (Hot Folder Pattern & Decoupling)
- **情境 (Situation)**：工廠原有的 ERP 系統老舊且無對外 API，WMS 需取得其領料與入庫單據，且不能影響 WMS 本身極其頻繁的行動端掃描連線效能。
- **行動 (Action)**：
  - 放棄高風險的資料庫直連，導入「檔案交換 (Hot Folder)」微服務架構。撰寫獨立的 Node.js 背景巡邏服務 (`watcher.js`)。
  - 嚴格界定系統邊界：ERP 只管總帳與開單，WMS 掌控實體儲位與執行人員。實作中介資料表 (`erp_pending_tasks`) 以非同步方式處理單據，並實施以單號為唯一索引的冪等性 (Idempotency) 防重複讀取機制。
- **結果 (Result)**：達成完美的系統解耦。即便 ERP 伺服器晚間關機維護，夜班廠房的 WMS 依舊能高速進行實體收發料。隔日清晨由 WMS 一鍵匯出符合 ERP 定規模板的 Excel，實現內勤人員「零人工對帳、一鍵匯入過帳」的全自動化迴圈。

### 2. 行動裝置非同步渲染競爭與資源崩潰修復 (Race Condition & Memory Leak Fix)
- **情境 (Situation)**：將手機鏡頭作為掃描工作站時，作業員從「掃描成功扣帳」到系統「重置畫面準備下一掃」的高頻切換中，極易引發 iOS/Android 瀏覽器當機白屏。
- **行動 (Action)**：
  - 診斷為典型的 React 並發競爭問題：在舊的連續掃描中，相機套件 (`html5-qrcode`) 來不及釋放硬體串流資源，React 就已經將負責掛載鏡頭的 `#reader` DOM 元素移除重建。
  - 我徹底解除了以按鈕 `onClick` 強制啟動鏡頭的舊模式，導入純狀態驅動 (`useState` 先行)。利用 `setTimeout` 延遲隊列與 `try...catch` 萬用攔截確保 DOM 絕對已經存在後，再呼叫第三方硬體 API 接管。
- **結果 (Result)**：打造出極為平滑的連續掃描體驗。作業員刷卡與刷料件的動作銜接流暢，不再發生任何閃退，徹底省下購置工業級 PDA 掃描槍數十萬台幣的設備預算。

### 3. 前端視覺演算法最佳化與 2D 疊貨渲染 (Frontend DOM Optimization)
- **情境 (Situation)**：系統需要渲染帶有數百個儲位網格的平面圖。當加入了各種複雜地形（大型走道、門口）且同一座標點存放多種物料（層架疊貨）時，畫面不僅跑版，底層 DOM 節點更是過多導致卡頓。
- **行動 (Action)**：
  - 放棄消耗記憶體的 Canvas 工具，改以原生 Tailwind CSS Grid 為底層。利用 React `useMemo` 動態計算極端座標，鎖定外框網格比例。
  - 開發自帶微語言（例如 `#V_#`）的正則解析器，合併大型走道的 DOM (span)。更撰寫了「座標重疊剔除演算法」，在繪製跨網格組件時，動態將底層被遮蔽的原生單元格設為 `null` 捨棄算繪，精簡超過 30% 渲染負擔。
  - 首創「座標聚合庫存徽章」：當偵測到一處有多筆庫存（如同座標但不同樓層或批號），利用絕對座標 (`absolute`) 於單元格右上角浮動聚合數量標籤，避免點擊死角。
- **結果 (Result)**：實現了純 HTML/CSS 驅動的複雜倉儲地圖，即使在低階平板也能流暢維持 60fps 體驗。並結合 `framer-motion` 產生如呼吸燈 (`animate-ping`) 的 BOM 表齊套取料引導動畫，極具視覺衝擊力。

### 4. 資料庫的事務原子性、軟刪除與極速部署 (ACID & Zero-Config Deployment)
- **情境 (Situation)**：工廠沒有專職維運工程師，且遭遇「大盤點」時需要一次傾倒數萬筆包含料號、安全庫存與地圖座標的初始資料，效能與防呆成為首要顧慮。
- **行動 (Action)**：
  - 大膽採用兼具高效與檔案型的 `SQLite` 搭配 Node.js 持久化；徹底捨棄消耗資源的 ORM 迴圈，純手工利用 `db.transaction()` 包裝萬筆資料的批次 `INSERT / UPDATE`。
  - 建置極為嚴格的「軟刪除 (Soft Delete)」與「人員軌跡紀錄 (Operator ID)」稽核機制。系統不執行硬刪除，而是透過單一事務操作翻轉 `is_deleted` 並同步回滾 `inventory` 表，讓每筆防呆退回皆有跡可循。
- **結果 (Result)**：
  - **效能面**：將一萬多筆聯合資料匯入時間從數分鐘壓縮至 **1~2 秒內**，且過程遭遇任何錯誤將完美 Rollback 不留殘渣。
  - **維運面**：撰寫智慧型 `.bat` 批次檔，利用動態探測自動尋找本機 Node.js 路徑，創造「隨插即用、點兩下即起飛」的零配置部署 (Zero-Config)，完美契合傳統產線的無痛導入需求。
