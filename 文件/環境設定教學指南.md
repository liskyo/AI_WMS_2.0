 # 各式專案環境設定教學指南 (新人手冊)

這份手冊旨在幫助新進人員快速判斷專案類型，並完成對應的開發環境設定。

## 1. 如何判斷專案類型 (特徵檔案判斷法)

在進入一個新專案時，請優先觀察根目錄下的**特徵檔案**，這能幫助你決定使用哪套工具。

### A. Node.js (JavaScript/TypeScript)
*   **特徵檔案**: `package.json`, `package-lock.json`, `yarn.lock`, `pnpm-lock.yaml`
*   **判斷基準**:
    *   如果有 `yarn.lock` -> 必須使用 **Yarn**
    *   如果有 `pnpm-lock.yaml` -> 必須使用 **PNPM**
    *   只有 `package-lock.json` -> 使用 **NPM**

### B. Python
*   **特徵檔案**: `requirements.txt`, `environment.yml`, `pyproject.toml`
*   **判斷基準**:
    *   如果有 `environment.yml` -> 使用 **Conda**
    *   如果有 `requirements.txt` -> 使用 **venv (內建虛擬環境)**
    *   如果有 `pyproject.toml` -> 使用 **Poetry**

### C. Docker (容器化)
*   **特徵檔案**: `Dockerfile`, `docker-compose.yml`
*   **判斷基準**: 看到這些檔案，代表專案應運行在 **Docker 容器**內。

## 2. 環境設定流程與終端機指令

### 【Node.js 專案】
1.  **安裝與產生檔案**:
    *   `npm install` (產生 `node_modules/`, `package-lock.json`)
    *   `yarn install` (產生 `node_modules/`, `yarn.lock`)
2.  **常用啟動指令**:
    *   `npm run dev` (前端 React/Vue 常見)
    *   `node server.js` (後端啟動)

### 【Python 專案 (venv)】
1.  **建立與啟動虛擬環境**:
    ```powershell
    python -m venv venv
    # 啟動 (Windows)
    .\venv\Scripts\activate
    ```
2.  **安裝依賴**:
    ```powershell
    pip install -r requirements.txt
    ```
3.  **確認環境**:
    *   確認終端機開頭出現 `(venv)` 字樣。

### 【Python 專案 (Conda)】
1.  **建立環境**:
    ```powershell
    conda env create -f environment.yml
    ```
2.  **啟動環境**:
    ```powershell
    conda activate [設定檔內的名稱]
    ```

### 【Docker 專案】
1.  **啟動服務**:
    ```powershell
    docker-compose up -d
    ```
2.  **進入容器調試**:
    ```powershell
    docker exec -it [容器名稱] /bin/sh
    ```

## 3. 環境變數 (.env) 管理規範

**這是一件非常重要的安全規範：**

1.  **檔案名稱與流程**:
    *   `.env.example`: **範本檔** (由資深同事維護，需進 Git)。
    *   `.env`: **設定檔** (由每個人手動建立，**絕對禁止進入 Git**)。
2.  **設定步驟**:
    *   新人必須將 `.env.example` 複製一份並重新命名為 `.env`。
    *   根據專案提示填入正確的 `DB_URL`、`API_KEY` 等資訊。
    *   執行指令: `cp .env.example .env` (或手動複製)。

## 4. 懶人判斷流程 (流程圖描述)

1.  **Step 1**: 觀察檔案 -> 是 Node (看 `package.json`) 還是 Python (看 `requirements.txt`)？
2.  **Step 2**: 建立環境 -> `npm install` 或是 `python -m venv venv`。
3.  **Step 3**: 設定變數 -> 複製 `.env.example` 為 `.env`。
4.  **Step 4**: 啟動 -> 執行啟動腳本。

## 5. 多環境配置 (本機測試 vs Server 佈署)

非常常見！一個專案通常會區分**本機開發環境 (Local/Development)** 與**伺服器正式環境 (Production)**。

### A. 透過環境變數區分
*   `.env.development`: 本機開發用 (連線到本地資料庫)。
*   `.env.production`: 伺服器佈署用 (連線到正式資料庫)。
*   **啟動指令差異**:
    *   本機: `npm run dev` (預設讀取開發環境變數)
    *   Server: `NODE_ENV=production node server.js` (強制指定為正式環境)

### B. 透過 Docker 設定檔區分
*   `docker-compose.yml`: 本機開發用 (開啟除錯工具、掛載本地資料夾以便熱更新)。
*   `docker-compose.prod.yml`: 伺服器正式佈署用 (優化效能、設定當機自動重啟 `restart: always`)。
*   **啟動指令差異**:
    *   本機: `docker-compose up -d`
    *   Server: `docker-compose -f docker-compose.prod.yml up -d`
