# 智慧倉儲動態管理系統 2.0 - 環境設定教學指南

這份指南專門介紹「智慧倉儲動態管理系統 2.0」的架構，並說明**本機開發 (Local)** 與**伺服器正式佈署 (Production)** 兩種不同情境的環境啟動方式。

---

## 1. 專案架構概覽

本專案採用前後端分離，並完全容器化 (Docker)。
*   **前端 (`client/`)**: 使用 Vite + React/Vue (Node.js)。
*   **後端 (`server/`)**: 使用 Node.js + SQLite (`warehouse.db`)。
*   **環境共用**: 包含 `.env` 設定檔、與 Docker 管理配置檔。

---

## 2. 拿到專案第一步 (無論是哪種環境)

1.  **安裝必備軟體**:
    *   [Git](https://git-scm.com/) (版本控制)
    *   [Docker Desktop](https://www.docker.com/products/docker-desktop/) (容器執行環境)
2.  **取得程式碼**:
    ```powershell
    git clone [專案Git網址]
    cd 智慧倉儲動態管理系統2.0
    ```
3.  **環境變數準備**:
    *   由於 `.env` 不會進入版控，請向專案負責人索取對應的 `.env` 檔案格式，並放置於專案根目錄。

---

## 3. 情境 A：本機開發 (Local Development)

本機開發的宗旨在於「方便修改程式碼並即時預覽」。

*   **使用工具**: `docker-compose.yml`
*   **特色**: 
    1. 掛載了 `./server` 與 `./client` 資料夾，每次改動程式碼會即時熱更新 (Hot Reload)。
    2. 開放雙 Port (前端 `5173`，後端 API `3000`) 方便除錯。
    3. 自動同步讀取 `./20260210 料架QRcode.xlsx` 測試資料。

**啟動流程**:
1.  打開終端機，確認位於專案根目錄 (`智慧倉儲動態管理系統2.0/`)。
2.  執行 Docker 啟動指令：
    ```powershell
    docker-compose up -d
    ```
3.  **如何停止**:
    ```powershell
    docker-compose down
    ```

> 💡 **快速腳本**：您也可以直接點擊根目錄的 `run_locally.bat` 來快速啟動本機測試。

---

## 4. 情境 B：伺服器佈署 (Production Server)

伺服器佈署的宗旨在於「效能、穩定、安全性」。

*   **使用工具**: `docker-compose.prod.yml`
*   **特色**:
    1. 不掛載原始碼，而是將程式完全封裝進名為 `wms-app-prod` 的 Image (映象檔) 中。
    2. 強制載入 `NODE_ENV=production` 環境變數，啟動最高效能模式。
    3. 只掛載必要的資料檔 (`warehouse.db`) 與上傳資料夾 (`uploads/`)。
    4. 具有崩潰自動重啟機制 (`restart: unless-stopped`)。

**佈署/更新流程**:
1.  將最新的程式碼 Pull 到 Server：
    ```bash
    git pull
    ```
2.  執行更新與啟動指令：
    ```bash
    docker-compose -f docker-compose.prod.yml up -d --build
    ```
    *(加上 `--build` 是為了確保程式碼更新後，Docker 會重新打包最新版本的 `wms-app-prod` 映象檔)*
3.  **如何停止**:
    ```bash
    docker-compose -f docker-compose.prod.yml down
    ```

## 5. 常見問題 (Q&A)

**Q1：為什麼我本機修改了程式碼，畫面卻沒變？**
> 請確認您目前使用的是 **情境 A (`docker-compose.yml`)**。若誤用了 **情境 B**，程式碼是被包死的，必須重新 `build` 才會生效。

**Q2：專案有提供快速測試的腳本嗎？**
> 有的，根目錄下有 `run_test.bat` 可以執行相關的測試腳本。

**Q3：我的資料庫檔案在哪裡？**
> 核心資料庫為根目錄或 server 目錄下的 `warehouse.db` (SQLite)，在上 Server 正式佈署時，此檔案將會被掛載至容器內持久化保護。
