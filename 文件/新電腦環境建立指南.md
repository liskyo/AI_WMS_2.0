# 新電腦環境建立指南 (從零開始)

當你拿到一台全新的電腦，或是需要在新設備上運行專案時，請依照以下步驟進行基礎建設。

## 1. 必備基礎工具安裝

在執行任何專案指令前，請確保電腦已安裝以下核心工具：

### A. 版本控制 (Git)
*   **用途**: 下載與管理程式碼。
*   **指令檢查**: `git --version`
*   **下載網址**: [git-scm.com](https://git-scm.com/)

### B. 執行環境 (依專案需求)
根據你的專案類型，至少安裝其中一項：
1.  **Node.js**: [nodejs.org](https://nodejs.org/) (建議安裝 LTS 版本)。
    *   檢查: `node -v`
2.  **Python**: [python.org](https://www.python.org/)。
    *   檢查: `python --version`
3.  **Docker Desktop**: [docker.com](https://www.docker.com/) (若專案有 Dockerfile)。

---

## 2. 專案建立流程

### Step 1: 複製專案 (Clone)
打開終端機 (PowerShell 或 Git Bash)，執行：
```powershell
git clone [專案遠端網址]
cd [專案資料夾名稱]
```

### Step 2: 基礎判斷與安裝
此時請參考 [環境設定教學指南.md](./環境設定教學指南.md) 進行判斷。
*   **如果是 Node.js**: 執行 `npm install`。
*   **如果是 Python**: 建立 venv 並執行 `pip install`。

### Step 3: 配置環境變數 (.env)
這通常是新電腦最容易漏掉的一步：
1.  找到 `.env.example`。
2.  複製並更名為 `.env`。
3.  **請向專案負責人詢問**最新的開發金鑰或資料庫密碼，並填入 `.env`。

---

## 3. 常見問題排除 (Quick Fix)

*   **問題**: `npm` 不是內部或外部命令。
    *   **解法**: Node.js 未安裝或未重啟終端機。
*   **問題**: `Execution Policy` 權限錯誤 (Windows)。
    *   **解法**: 以管理員身份執行 `Set-ExecutionPolicy -ExecutionPolicy RemoteSigned`。
*   **問題**: Git Clone 失敗 (Permission Denied)。
    *   **解法**: 檢查 SSH Key 是否已加入 GitLab/GitHub。

---

## 4. 總結 Check-list
- [ ] Git 已安裝且登入
- [ ] Node.js / Python 已安裝
- [ ] 專案已 Clone 到本地
- [ ] 依賴套件已安裝完成 (`node_modules` 或 `venv` 已產生)
- [ ] `.env` 已正確配置
- [ ] 執行啟動指令 (如 `npm run dev`)
