# 智慧倉儲 2.0 - 伺服器初次部署新手指南 (SOP)

本手冊專為初學者設計，旨在以最簡單、明確的步驟引導您完成從「本機開發端」到「雲端/地端伺服器」的部署過程。

## 📋 部署前準備清單
1.  **伺服器資訊**：正式環境 IP、登入帳號與密碼（或 SSH Key）。
2.  **網路環境**：確保 MIS 已開放伺服器 **Port 3000**（本系統專用進入點）。
3.  **必備工具**：本機已安裝 PowerShell 或終端機。

---

## 🚀 第一階段：伺服器環境初始化
連線至伺服器後，執行以下指令（一行一行執行，括號內是說明）：

```bash
# 1. 建立並進入專案目錄
mkdir -p ~/wms_system
cd ~/wms_system

# 2. 建立資料持久化目錄 (避免重啟後資料遺失)
mkdir -p backups uploads

# 3. 檢查資料庫狀態
# 如果您「沒有」舊資料庫要帶上去，請執行：
touch warehouse.db
# 如果您「已經有」準備好的資料庫檔案，請跳過此行，直接看下一階段。
```

---

## 📦 第二階段：檔案傳輸 (在本機執行)
在本機電腦的專案資料夾內點擊滑鼠右鍵，選擇「在終端機中開啟」或「開啟 PowerShell」：

```powershell
# 執行以下指令 (將 192.168.x.x 換成實際伺服器 IP)
# 說明：這會將專案除了 node_modules 以外的所有檔案同步到伺服器
scp -r * ws@192.168.x.x:~/wms_system
```

---

## 🛠️ 第三階段：啟動系統 (在伺服器執行)
回到伺服器視窗，執行啟動命令：

```bash
cd ~/wms_system

# 使用 Docker Compose 進行自動建置並在背景執行
docker-compose -f docker-compose.prod.yml up -d --build
```

### ✅ 如何確認啟動成功？
輸入 `docker-compose -f docker-compose.prod.yml ps`
*   **狀態顯示 `Up` 或 `Running`**：恭喜您，系統已成功運行！
*   **如何登入？**：瀏覽器輸入 `http://伺服器IP:3000`
    *   預設帳號：`admin`
    *   預設密碼：`admin123`

---

## 🔍 第四階段：常見問題排解 (Debug)

如果您遇到問題，請對照以下情況進行排除：

### 1. 網頁打不開 (無法連線)
*   **原因 A**：伺服器防火牆沒開（Port 3000）。
    *   **檢查**：在伺服器輸入 `sudo ufw status` (若是 Ubuntu) 確認 3000 是否 Allowed。
*   **原因 B**：3000 號連接埠被其他程式佔用了。
    *   **檢查**：輸入 `netstat -tulpn | grep 3000`，若有東西在跑，請聯繫 MIS 詢問。

### 2. 系統有畫面，但登入顯示「資料庫錯誤」
*   **原因**：這通常發生在 `warehouse.db` 權限不足或檔案損壞。
    *   **解決**：輸入 `ls -l warehouse.db` 確保權限非 Root 獨佔，或嘗試重啟服務：
        `docker-compose -f docker-compose.prod.yml restart`

### 3. 如何查看錯誤訊息 (看日誌)？
如果您發現系統運作異常，請輸入以下指令查看即時的跑碼訊息：
`docker-compose -f docker-compose.prod.yml logs -f app`
*   *提示：按下 `Ctrl + C` 可退出查看。*

### 4. 檔案傳輸時失敗 (Permission Denied)
*   **原因**：SSH 連線逾時或權限不足。
    *   **解決**：確認 IP 是否正確，並確保傳輸時沒有程式（如本地 Server）正在佔用該檔案。

---

## 🔍 第五階段：數據備份與維護

為了確保數據安全，系統內建了自動備份機制，部署人員只需知悉以下運作邏輯：

1.  **自動備份時間**：每天 **凌晨 02:00** 會自動執行資料庫快照。
2.  **備份檔位置**：伺服器上的 `~/wms_system/backups` 目錄。
3.  **自動清理機制**：系統會自動保留 **最近 7 天** 的備份檔，並清理更久的舊檔，以節省伺服器空間。
4.  **手動備份方法**：若有重大維護前想手動備份，請在伺服器專案目錄下執行：
    `cp warehouse.db backups/manual_backup_$(date +%Y%m%d).db`
    *(註：這會將資料庫快照直接存入備份資料夾中。)*

5.  **下載回開發電腦保存 (在本機執行)**：
    若想將伺服器的備份抓回本地電腦存檔，請在 **本機 PowerShell** 執行：
    `scp ws@192.168.x.x:~/wms_system/backups/*.db "C:\Users\sky.lo\Desktop\智慧倉儲動態管理系統2.0\server\backups\"`
    *(註：這會將伺服器 backups 資料夾內所有的備份檔，通通抓回您本機指定的備份資料夾中。)*

---

## 📝 後續更新與維護小訣竅
以後只要程式碼有改動，只需重複 **「第二階段 (上傳)」** 與 **「第三階段 (啟動)」** 即可。
系統會自動覆蓋舊程式，但您的資料庫 (`warehouse.db`) 與備份目錄 (`backups`) 會因為掛載機制被保留，更新不會影響既有資料。
