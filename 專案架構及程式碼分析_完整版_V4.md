## 一、 系統巨觀架構與技術棧 (Architecture & Tech Stack)

本專案採用經典的**前後端分離（Client-Server）架構設計**，並以「極輕量、易部署、高效能、高整合度」為四大哲學，屏除傳統製造業常見的笨重套裝軟體與關聯式大廠資料庫。

### 1. 前端 (Client - 倉儲人員與管理端 UI)
*   **核心框架**: `React.js v18` (搭配 Vite 建置，提供極速 HMR 開發體驗)
*   **介面樣式庫**: `TailwindCSS v3` (Utility-first CSS，負責高度客製化的響應式佈局與組件庫)
*   **路由管理**: `react-router-dom v6` (SPA 單頁面應用路由控制)
*   **狀態與 API 溝通**: `Axios` (封裝攔截器與 Token 驗證管理)
*   **動畫與互動**: `framer-motion` (處理地圖閃爍導航 Ping、流暢的視窗轉場、清單展開)
*   **硬體設備串接**: `html5-qrcode` (呼叫手機 WebRTC 相機 API 掃描 1D/2D 條碼)
*   **圖表與報表**: 支援匯出 `Excel`，利用原生的 Blob 與 HTML Table 轉換技術。

### 2. 後端 (Server - 邏輯運算與資料 API)
*   **核心框架**: `Node.js` + `Express.js` (RESTful API 伺服器)
*   **持久化資料庫**: `better-sqlite3` (捨棄 MySQL，採用同步、超低延遲且支援 ACID 的檔案型 SQLite，使部署免安裝 DB。配合 Node Event Loop 特性，讀寫效能極高)
*   **系統整合 (Integration)**: 內建 `chokidar` 檔案監控引擎，支援 ERP 資料夾交換同步機制。
*   **身分與權限驗證**: 輕量級 Base64 JSON Payload (不依賴龐大的 JWT，自建簡易 Token，實現無狀態驗證)
*   **檔案上傳/解析**: `multer` 搭配 `xlsx` 庫，負責匯入地圖坐標格網、庫存與料件主檔資料。

## 三、 資料庫 Schema 演進與核心關聯 (Data Modeling)

系統透過嚴謹的正規化關聯，解決實體倉儲常遇到的「一物多位」、「一位多物」以及「多系統資料拋轉」等複雜的狀態管理。

### 1. `items` (料件規格主檔)
*   **欄位結構**: `id` (PK), `barcode` (庫存料號), `name`, `description`, `category`, `unit`, `safe_stock` (安全存量)。
*   **安全庫存引擎**: 支援獨立防呆，未填寫或舊資料強制設定為 `0`。支援前端就地即時編輯，是觸發 Dashboard 警告與防護斷鏈的基準數值。

### 2. `locations` (儲位物理主檔與空間映射)
*   **欄位結構**: `id` (PK), `code` (儲位代碼), `x` (X座標), `y` (Y座標), `type` (儲位特性)。
*   **V4 地形引擎升級**: 搭配 `Excel` 最新解析邏輯，儲位不只是一個文字。系統透過座標解析，能將原本空白、無編號的區域判定為「障礙物/不可通行區」，甚至能判斷高層架疊起時的視覺狀態，反映到 2D 網格的 DOM 生成規則。

### 3. `inventory` (動態庫存表 - 核心樞紐)
*   **欄位結構**: `id` (PK), `item_id` (FK), `location_id` (FK), `quantity` (當下庫存量), `updated_at`。
*   **特性設計**: 當數量被清零時，不會被 `DELETE`，藉以保存這格儲位**曾經存過誰**的線索。在 V4 報表生成中，透過 `GROUP BY location_id` 與子查詢字串化 (GROUP_CONCAT)，可以完美在一格欄位內列出該儲位的「多種混放料件」，極大化報表閱讀性。

### 4. `transactions` (異動歷史軌跡與權責劃分表)
*   **V4 架構大升級 - 職責歸屬**: 
    新增了針對操作者的欄位（如 `operator_id`）。系統清楚劃分了「ERP 開單人(created_by)」與「WMS 實際扣帳人(executed_by)」。配搭前端強制刷卡機制，使每筆庫存進出具備不可否認性，徹底解決帳務稽核難題。

### 5. `erp_pending_tasks` (ERP 對接中介表 - 新增)
*   **設計理念**: 為避免 ERP 系統頻繁斷線或鎖表影響 WMS 效能，利用此表中繼存放背景 `watcher.js` 吃進來的 Excel 拋轉單據。待現場人員完成實體撿料後，更新狀態並由 WMS 定期產出確認檔供 ERP 匯入過帳。

## 四、 深度技術亮點分析 (Engineering Highlights)

這裡整理了本專案為了解決效能、硬體限制與企業整合所實作的**五大頂級工程架構**：

### 1. 異質系統的高容錯非同步對接 (Hot Folder x Watcher Pattern)
針對傳統沒有現代化 API 的老舊 ERP（如鉅茂、鼎新舊版），系統捨棄了高成本的資料庫直連，改以「檔案交換」實踐高解耦度：
*   **無依賴獨立運行**: WMS 後端透過 Node.js 的 `chokidar` 持續監聽指定的網路共用資料夾。當生管匯出單據 Excel 並丟入資料夾的瞬間，WMS 背景攔截、解析並寫入 `erp_pending_tasks`，隨即將原檔案搬離原處預防重複讀取 (Idempotency)。
*   ERP 死機、網路中斷，完全不影響現場收發料；而 WMS 在現場完成盤點後吐出的 Excel，設計上 100% 吻合 ERP 端內建的「批次匯入 Template 格式」，達成免人工轉換、無縫拋帳。

### 2. 進化的 2D 視覺化儲位引擎 (Visual Engine - 疊貨與地形渲染)
為解決前端效能瓶頸，本專案的地圖核心 `MapGrid.jsx` 不依賴龐大的 Canvas 或 Three.js，純透過 React DOM 控制：
*   **動態邊界探勘 (`useMemo` 優化)**: 載入座標時，動態算出極端值的 `maxX` 與 `maxY`，利用 inline-style 賦予父容器 CSS Grid 屬性，實現精準定位不跑版。
*   **多料疊貨標籤設計**: 當多種物料混放於同一個 `(x, y)` 座標時（例如料架有上下層），系統會自動聚合總量，並利用 Tailwind 的 `absolute` 標籤於單元格角落浮現醒目的提示（例如右上角紅底白字的總數標籤），徹底解決視角重疊的顯示 Bug。
*   **髒點清除更新**: 在 V4 開發脈絡中，徹底排除了舊版資料重新匯入時導致地圖右下角出現多餘字元 ('A') 的資料殘留幽靈物件。

### 3. 主動防禦型安全庫存與採購建議系統 (Active Defense Alert)
將系統從「被動填表」升級為「智能顧問」：
*   在伺服器端高效利用 SQL `SUM(quantity)` 對比 `items.safe_stock`。只要全場實體數量低於或觸及安全水位，系統不用等報表產出，會即時在前台首頁觸發鮮紅警告區。
*   點擊警告區跳轉，前端透過 URL 參數路由至報表模組，自動為主管篩選出「待補貨料件」，並一鍵使用原生 JS Blob 技術打包匯出 `xlsx`，直接變成給採購單位發信用的「建議採購 Excel 單」，消滅人工比對時間。

### 4. 嚴密的行動端掃描與非同步資源解鎖 (Mobile Safari Race Condition Fix)
在不花大錢投資硬體手持 PDA 的情況下，透過員工自身手機進行入庫掃碼：
*   **Https 破窗**: 透過 Vite 掛載 Basic SSL，打破 iOS Safari 與 Android 對 WebRTC API (相機捕捉) 的嚴格封鎖策略。
*   **生命週期管理黑魔法**: 當人員掃描條碼並觸發資料庫交易後，React 會執行極快速的畫面清理。為了避免前一次相機資源 (`html5-qrcode` instance) 來不及關閉而與 React 重渲染產生致命的 Memory Leak 或黑畫面當機，我將相機掛載點徹底依賴狀態機 (`useState`) 隔離控制。
*   這套「防呆、可重掃、絕不崩潰」的非同步解鎖流程，使得產線即刻進入高頻率掃描工作流也不致閃退。

### 5. 高吞吐盤點事務與零死角維運 (Batch Transactions & Zero-Config Deployment)
*   **SQLite 極限榨取**: 本系統匯入上萬筆原始基礎資料或地圖陣列時，藉由 SQLite `.transaction()`，避開 ORM 的迴圈等待。匯入一萬筆複合資料時間被壓制在 **1秒左右**。任何一行資料夾帶髒格式，整包資料完美 Rollback，保證資料庫的 ACID 絕對純淨。
*   **環境智能獵犬**: IT 工具 `run_test.bat` 會動態爬梳、自動尋找這台工業電腦內藏的 `node.exe` 絕對路徑，寫入本機快取。此機制的出現，讓傳統產線不再需要懂任何「系統變數設定」或「CMD 指令」。點兩下一鍵雙開前後端伺服器。
